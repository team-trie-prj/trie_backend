# 핵심 기능

---

## 기능1

**이미지+자연어 통합 검색 (VLM 기반)**  

- 기업 핵심 솔루션 도메인(도로 파손, 스마트 팩토리 등) 대상
- 이미지 데이터 + 자연어 질의 동시 입력 처리
- VLM 기반으로 복합 질의(예: “비 오는 날 발생한 포트홀 이미지”)의 의도/시각 맥락 해석
- 관련 사내 문서 및 공공데이터 등 외부 데이터와 통합 매칭 후 결과 제공

## 기능2

**AI 에이전트 기반 공공데이터·사내문서 자율 라우팅 검색 (하이브리드 RAG)**  

- LLM 기반 에이전트가 사용자 질의 분석
- 키워드 검색(BM25) vs 벡터 임베딩 검색 중 최적 방식 판단(라우팅) 및 수행
- 도로 시설물·산업안전·교통 등 도메인 사내 지식 + 공공 통계/API 교차 탐색
- 민원·법률 등 모호한 질의는 사전 정의 ‘맞춤형 질의 템플릿’ 제시로 정확도 확보

## 기능3

**검색 결과 기반 실무 맞춤형 보고서 자동 생성**  

- 하이브리드 검색으로 수집된 다중 출처 데이터(사내 매뉴얼, 현장 점검 이미지, 공공데이터 통계 등) 종합
- 목적별 보고서(점검 일지, 민원 대응 브리핑 등) 초안 자동 생성
- 실무 즉시 활용 가능한 형태로 제공

# 문제 정의

---

## 문제요약

- 데이터 파편화
- 단순 키워드 검색 중심 체계의 한계
- 기존 시스템 아키텍처 불안정(오류 및 비용 부담)
- 상기 요인으로 인한 실무 업무 비효율

## 해결방안

- 목표: 오렌지파이 5 Plus 기반 소규모 자체 서버 환경 최적화
- 방향: **‘멀티모달 및 에이전트 기반 하이브리드 RAG 통합 시스템’**으로 전면 재설계
- **에이전트 기반 질의 분석 및 라우팅**
    - 사용자 질의 의도·도메인·데이터 유형(사내 문서·이미지·공공데이터) 분해
    - BM25 키워드 검색 / 벡터 의미 검색 / 공공 API 호출 중 최적 경로 선택
- **모호한 질의 템플릿 제시**
    - 민원 대응, 법률/규정 확인 등 고난도 질의 식별
    - 필수 입력 항목 포함 템플릿 제시 → 재질의 유도
    - 검색 실패(Zero-hit) 최소화 및 정확도 확보
- **VLM 기반 멀티모달 검색**
    - 현장 이미지(포트홀, 안전장비 미착용, 설비 이상 등) 해석
    - 텍스트/메타데이터 결합 검색
    - 근거(참조 문서, 유사 사례) 동시 제공
- **실무 보고서 자동 생성**
    - 검색 결과(사내 매뉴얼, 점검 이미지, 통계 수치) 구조화
    - 목적별 보고서 초안 자동 생성(점검 일지, 민원 브리핑, 개선 권고안 등)
- **리소스 제약 대응 아키텍처**
    - 고비용 추론: 외부 LLM/VLM API 위임
    - 로컬 서버: 인덱싱·메타데이터·캐시·권한·서빙 집중
    - 3주 내 구현 가능성과 운영 안정성 동시 확보

# 유사 서비스 분석

---

| 대표 서비스 | 특징 | 한계 |
| --- | --- | --- |
| 공공데이터포털 및 사내 검색엔진 (Elasticsearch 등) |   • 공공기관의 방대한 원천 데이터(API, CSV) 카탈로그 통합 제공
  • 문서 내 키워드 색인 기반의 빠른 텍스트 검색 속도 보장 |   • 단순 텍스트 매칭으로 사용자의 '질의 의도(Context)'와 문맥을 파악하지 못해 검색 정확도 저하
  • 현장 사진 등 이미지 기반 검색이 불가하며, 여러 데이터를 융합해 보고서로 만들어주는 지능형 분석 기능 부재 |
| 범용 대화형 AI 챗봇 (ChatGPT, Claude 등) |   • 뛰어난 자연어 이해력을 바탕으로 문서 요약 및 자연스러운 질의응답 지원
  • 다양한 형태의 문서 포맷팅 및 번역 능력 탁월 |   • 기업의 사내 비공개 운영 데이터나 실시간 공공데이터 API와 직접 연동되지 않아 환각(Hallucination) 현상 발생 우려가 매우 큼
  • 지속적인 클라우드 트래픽 의존 및 비용 발생 |

# 차별점

| 차별점 | 설명 | 기존 서비스 대비 효과 |
| --- | --- | --- |
| 차별점 1
**에이전트 자율 라우팅 + 템플릿 제시** |   • 질의 의도 분석 후 도구 선택(키워드/BM25, 벡터 검색, 공공 API 호출)
  • 모호 질의 식별 후 템플릿 제시 → 재질의 유도
  • 현장 실무자 관점 입력 부담 최소화 |   • 검색 실패(Zero-hit) 최소화
  • 정보 탐색 리드타임 단축
  • 비전문 사용자도 일상 질의만으로 사내·공공 데이터 융합 결과 확보 |
| 차별점 2
**저비용 엣지 인프라 + 멀티모달 보고서 자동화** |   • 오렌지파이 자체 서버 구동 전제 최적화 설계
  • 이미지(VLM) + 텍스트/메타데이터 결합 분석
  • 결과를 목적별 보고서 초안으로 구조화/변환 |   • 클라우드 운영 비용 절감
  • 검색 → 분석 → 문서 작성의 실무 파이프라인 자동화
  • 현장 업무 생산성 향상 |

# 도메인

## 도메인1 `RoadFacilityManagement`

---

- 도로 파손 AI 솔루션과 연계되는 도메인
- 현장 수집 포트홀/도로 균열 이미지 + 지자체 도로 유지보수 매뉴얼 + 공공데이터포털 민원/보수 통계 융합
- 시설물 상태 관리 및 보수 계획 수립 지원

### Entity1 `KnowledgeDocument`

- 사내 운영 매뉴얼 원본 텍스트 저장
- 검색 성능 향상을 위한 청크(Chunk) 단위 분할 텍스트 저장
- 임베딩 벡터(Vector) 및 색인 메타데이터 저장
- 공공데이터포털 수집 데이터(통계/API 응답) 메타데이터 통합 저장

### Entity2 `VisualResource`

- 현장 점검 이미지 원본 저장
- VLM 분석 결과로 생성된 이미지 맥락 텍스트(라벨/상황 묘사/추정 원인 등) 저장
- 이미지-문서-도메인 연계를 위한 메타데이터 저장(촬영 시각/위치/현장 구분 등)

### Entity3 `GeneratedReport`

- AI가 생성한 실무 문서 초안(점검 일지, 민원 브리핑 등) 저장
- 세션 단위 식별자 부여로 동일 결과 반복(캐시 문제) 방지
- 생성 시점의 입력 질의/참조 근거/사용 데이터 출처 메타데이터 저장

## 도메인2 `IndustrialSafetyManagement`

---

- 스마트 팩토리 점검 솔루션과 연계되는 도메인
- 산업 현장 점검 이미지 내 위험 요소 식별 + 사내 안전보건 매뉴얼 + 공공 중대재해 법령/가이드 데이터 결합
- 사고 예방 조치 및 점검 일지 작성 지원

### Entity1 `KnowledgeDocument`

- 안전보건 매뉴얼/점검 기준서/작업 절차서 등 사내 문서 저장
- 청크 분할 및 임베딩/색인 메타데이터 저장
- 공공 법령/가이드/행정 해석 데이터 메타데이터 저장

### Entity2 `VisualResource`

- 점검 이미지 원본 및 VLM 분석 맥락 텍스트 저장
- 위험 요소 라벨링/상황 요약/관련 규정 후보 등 파생 데이터 저장

### Entity3 `GeneratedReport`

- 점검 일지/개선 권고안/위험성 평가 요약 등 실무 문서 초안 저장
- 세션 단위 식별자 및 근거 데이터 출처 메타데이터 저장

## 도메인3 `TrafficManagement`

---

- 공공 CCTV 이상행동 탐지 솔루션과 연계되는 도메인
- 실시간 교통량/사고 다발 구역 등 공공데이터 API + 사내 교통 관제 분석 데이터 결합
- 교통 현황 파악 및 정책/운영 의사결정 지원

### Entity1 `KnowledgeDocument`

- 관제 분석 리포트/운영 매뉴얼/정책 자료 등 사내 문서 저장
- 청크 분할 및 임베딩/색인 메타데이터 저장
- 공공 교통 데이터(API/통계) 메타데이터 저장

### Entity2 `VisualResource`

- CCTV 스냅샷/현장 이미지(허용 범위 내) 및 VLM 분석 맥락 텍스트 저장

### Entity3 `GeneratedReport`

- 상황 브리핑/사고 대응 보고/정책 검토 메모 등 문서 초안 저장
- 세션 단위 식별자 및 근거 데이터 출처 메타데이터 저장

# 통합 검색 `search`

---

이미지와 자연어 질의를 동시에 입력받아 사용자의 복합 검색 의도를 파악하고, 사내 문서 및 공공데이터를 융합 탐색

### 멀티모달 질의 분석 및 통합 쿼리 생성

1. 시스템은 텍스트 질의 및 첨부 이미지를 결합 분석하여 하이브리드 통합 검색을 수행
2. 시스템 처리
    - 업로드된 이미지를 VLM(비전언어모델)으로 분석하여 상황 맥락을 텍스트로 추출
    - 추출된 이미지 설명과 사용자 자연어 텍스트를 결합하여 최종 검색 쿼리 생성
    - 생성된 쿼리를 기반으로 사내 문서(키워드/벡터) 및 공공데이터(API/통계) 탐색
    - 검색 세션 식별자 발급 및 매 검색마다 캐시 갱신 처리(동일 결과 반복 오류 방지 목적)
3. 처리 결과
    - **`성공`**
        - 이미지 맥락이 반영된 검색 결과가 반환
        - 사내 문서 결과와 공공데이터 결과가 통합 뷰로 제공
        - 세션 식별자가 결과에 포함
    - **`실패`**
        - 검색 결과가 반환되지 않거나 텍스트 기반 검색만 수행
        - 실패 사유 제공
            - 지원하지 않는 이미지 포맷 또는 용량 초과
            - VLM API 호출 실패 또는 응답 지연
            - 자체 서버 리소스 부족(CPU/RAM)으로 인한 처리 실패

# 라우팅 검색 `routing`

---

LLM 에이전트가 질의를 분석하여 키워드/벡터 검색 중 최적 방식을 자율 결정하고, 모호한 질의는 템플릿으로 보완

### 에이전트 자율 라우팅 및 템플릿 제시

1. 시스템은 질의 특성을 판단하여 적합한 데이터 소스 및 검색 방식을 매칭하거나 맞춤형 템플릿을 제시
2. 시스템 처리
    - LLM 에이전트가 텍스트 질의의 의도 및 도메인 분류
    - 명확한 키워드 매칭이 필요한 경우 RDBMS/BM25 기반 키워드 검색 지시
    - 문맥 기반 유사도 탐색이 필요한 경우 Vector DB 기반 임베딩 유사도 검색 지시
    - 질의가 모호하거나 필수 파라미터가 누락된 경우 검색 보류 후 맞춤형 질의 템플릿 제시
3. 처리 결과
    - **`성공`**
        - 최적화된 방식에 따라 검색된 데이터가 병합되어 제공
        - 또는 맞춤형 질의 템플릿이 표시되어 사용자가 추가 입력 가능
    - **`실패`**
        - 검색 및 라우팅 미실행
        - 실패 사유 제공
            - 자체 서버 DB 커넥션 연결 실패
            - LLM 에이전트 API 응답 지연(Timeout)
            - 외부 공공 API 호출 실패 또는 레이트리밋

# 보고서 생성 `report`

---

검색된 다중 출처 데이터(사내 문서, 이미지 분석 결과, 공공 통계 등)를 종합하여 실무에 즉시 활용 가능한 문서 초안을 생성

### 검색 결과 기반 실무 맞춤형 문서 작성

1. 시스템은 하이브리드 검색 결과를 기반으로 LLM을 활용하여 지정된 양식의 문서를 자동 조립
2. 시스템 처리
    - 최종 검색 문서 텍스트, 공공데이터 통계 수치, 이미지 분석 결과를 컨텍스트(Context)로 구성
    - 사용자가 선택한 도메인 및 보고서 타입(점검 일지, 민원 브리핑 등)에 따른 시스템 프롬프트 적용
    - LLM이 컨텍스트를 종합 분석 및 요약하여 Markdown 기반 보고서 텍스트 생성
    - 생성 결과를 세션 단위로 저장하고, 참조 근거 및 데이터 출처 메타데이터 함께 기록
3. 처리 결과
    - **`성공`**
        - 서식이 적용된 실무 맞춤형 보고서 초안 제공
        - 참조 근거(문서/데이터) 및 생성 세션 식별자 제공
    - **`실패`**
        - 보고서 생성 중단
        - 실패 사유 제공
            - 참조할 검색 결과 데이터 부족으로 종합 분석 불가
            - LLM 토큰(Context Window) 한도 초과
            - 외부 LLM API 응답 지연 또는 오류
